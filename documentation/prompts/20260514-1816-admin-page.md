# Admin page

Build a password-protected admin page for the Swichers basketball platform. The page replaces `/upload` and requires JWT authentication via a single admin password stored as a bcrypt hash in `.env`.

## Backend changes

### Auth module (`backend/src/auth/`)
- `AuthService.login(password)`: compare against `ADMIN_PASSWORD_HASH` (bcrypt), return signed JWT (24h expiry)
- `POST /auth/login`: public endpoint
- `JwtAuthGuard`: manual guard extracting `Authorization: Bearer <token>`, verifying with `JwtService.verify`
- `AuthModule` exports `JwtAuthGuard` and `JwtModule`; imported in `AppModule`
- Protect `POST /score-sheet/extract` with `@UseGuards(JwtAuthGuard)`
- Add `ADMIN_PASSWORD_HASH` and `JWT_SECRET` to `backend/.env`

### Game entity
- Add `score_a: number` and `score_b: number` (`@Column({ type: 'int', nullable: true })`) to `Game`
- `GamePersistenceService.persist()`: set `score_a`/`score_b` from `data.stats.home/away.totals.team.points`
- `TeamService.findOne()`: derive `points`, `points_against`, `win` directly from `score_a`/`score_b` (no more TeamStatRow join for scores)

### File endpoint (`backend/src/file/`)
- `FileService.findById(id)`: query by PK
- `GET /files/:id`: public, streams PDF via `fs.createReadStream().pipe(res)` with `Content-Type: application/pdf` and `Content-Disposition: inline`
- Register `FileController` in `FileModule`

### Game list + delete (in `GameModule`, not a separate AdminModule)
- `GameService.list(search?, page)`:
  - Join `team_a`, `team_b`, `group`, `group.championship`
  - ILIKE filter on team names if `search` provided
  - Order by `day DESC`, paginate 20/page
  - Secondary query to build `file_id` map: `fileRepo.find({ where: { game: { id: In(gameIds) } } })`
  - Returns `{ data: GameListItem[], total, page, pageSize }`
- `GameService.delete(id)`: transaction cascading GameOfficer → PlayerStatRow → CoachStatRow → TeamStatRow → File (DB + disk unlink) → Game
- `GET /games` decorated with `@SkipThrottle()` (global throttle is 10 req/60s; debounced search would hit it)
- `DELETE /games/:id` protected by `JwtAuthGuard`, returns 204

### Game findOne
- Query `FileRepository` for `findOne({ where: { game: { id } } })` and include `file_id: file?.id ?? null` in response

## Frontend changes

### Auth context (`frontend/src/contexts/`)
- `AuthContext.tsx`: `AuthProvider` with `token` (sessionStorage), `login(password)` → POST /auth/login, `logout()`
- `useAuth.ts`: separate file for `useAuth` hook (required by react-refresh/only-export-components lint rule)

### Admin page (`frontend/src/pages/Admin.tsx`)
- If unauthenticated: `LoginForm` with password input, calls `login()`, shows error on failure
- If authenticated:
  - `UploadSection`: PDF input → `POST /score-sheet/extract` with Bearer token
  - `GamesSection`: debounced search (300ms), paginated table (date, home/away teams linked to `/teams/:id`, score linked to `/games/:id`, competition name), PDF button per row, "Supprimer" button with confirmation modal, prev/next pagination
- Route `/upload` → `/admin`

### Nav (`frontend/src/components/Layout.tsx`)
- Remove "Importer un score" button
- Show "Admin" link only when `token !== null`

### Game page (`frontend/src/pages/Game.tsx`)
- Add `file_id: number | null` to `GameData` type
- Render "Voir le PDF →" link in meta row when `file_id` is non-null
